# SelfOps Data Model

## Database: PostgreSQL (`selfops`)

All tables use UUID primary keys generated by `uuid_generate_v4()`. All timestamps are stored as `TIMESTAMPTZ` (UTC).

---

## Enums

### `incident_status`
Lifecycle states for an incident:

| Value | Description |
|-------|-------------|
| `OPEN` | Newly created, not yet processed |
| `ENRICHING` | Worker is collecting metrics and logs |
| `ANALYZING` | LLM analysis is in progress |
| `ACTION_REQUIRED` | Analysis complete, awaiting operator decision |
| `REMEDIATING` | A remediation action is executing |
| `MONITORING` | Remediation done, watching for recovery |
| `RESOLVED` | Incident is fully resolved |
| `FAILED_REMEDIATION` | Remediation action failed |
| `CLOSED` | Manually closed by operator |

### `severity_level`
| Value | Description |
|-------|-------------|
| `critical` | Service is down or data loss risk |
| `warning` | Degraded performance or approaching limits |
| `info` | Informational, no immediate action needed |
| `unknown` | Not yet determined |

### `action_status`
| Value | Description |
|-------|-------------|
| `PENDING` | Queued, not yet started |
| `RUNNING` | Ansible playbook executing |
| `SUCCESS` | Completed successfully |
| `FAILED` | Failed with error |

### `execution_mode`
| Value | Description |
|-------|-------------|
| `manual` | Triggered by human operator |
| `auto` | Triggered automatically by the system |

## Tables

### `incidents`
Central table. One row per unique infrastructure incident.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK, default uuid_generate_v4() | Unique identifier |
| `title` | TEXT | NOT NULL | Human-readable incident title |
| `status` | incident_status | NOT NULL, default 'OPEN' | Current lifecycle state |
| `severity` | severity_level | NOT NULL, default 'unknown' | Severity level |
| `service_name` | TEXT | nullable | Name of the affected service |
| `namespace` | TEXT | nullable | Kubernetes namespace |
| `environment` | TEXT | default 'production' | Deployment environment |
| `fingerprint` | TEXT | UNIQUE | Hash of alert labels — used for deduplication |
| `first_seen_at` | TIMESTAMPTZ | NOT NULL, default NOW() | When the incident was first detected |
| `last_seen_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Most recent alert for this incident |
| `resolved_at` | TIMESTAMPTZ | nullable | When the incident was resolved |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Row creation time |
| `updated_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Last row update time |

**Indexes:** `status`, `created_at DESC`, `fingerprint`

---

### `alert_events`
Raw alert payloads from Alertmanager. Multiple alerts can belong to one incident (via deduplication on fingerprint).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `incident_id` | UUID | FK → incidents(id) ON DELETE CASCADE | Parent incident |
| `alert_name` | TEXT | NOT NULL | Prometheus alert name |
| `fingerprint` | TEXT | NOT NULL | Alert fingerprint from Alertmanager |
| `labels` | JSONB | default '{}' | Alert labels (pod, namespace, severity, etc.) |
| `annotations` | JSONB | default '{}' | Alert annotations (summary, description) |
| `starts_at` | TIMESTAMPTZ | nullable | When the alert condition first triggered |
| `ends_at` | TIMESTAMPTZ | nullable | When the alert resolved (null if still firing) |
| `raw_payload` | JSONB | nullable | Complete raw webhook payload |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Row creation time |

**Indexes:** `incident_id`

---

### `incident_evidence`
Enrichment data collected by the worker. Stores metrics snapshots and log extracts.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `incident_id` | UUID | FK → incidents(id) ON DELETE CASCADE | Parent incident |
| `evidence_type` | TEXT | NOT NULL, CHECK IN ('metric', 'log', 'alert', 'analysis_input') | Type of evidence |
| `content` | JSONB | NOT NULL | The evidence payload (varies by type) |
| `captured_at` | TIMESTAMPTZ | NOT NULL, default NOW() | When this evidence was collected |

**Evidence content shapes:**
- `metric`: `{"query": "...", "result": [...prometheus_result...]}`
- `log`: `{"lines": ["...", "..."], "query": "...", "time_range": "..."}`
- `alert`: `{"alert_name": "...", "labels": {...}, "annotations": {...}}`

**Indexes:** `incident_id`

---

### `analysis_results`
LLM analysis output for an incident. One record per analysis run (can have multiple if re-analyzed).

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `incident_id` | UUID | FK → incidents(id) ON DELETE CASCADE | Parent incident |
| `model_provider` | TEXT | NOT NULL | LLM provider (e.g. "openrouter") |
| `model_name` | TEXT | NOT NULL | Model used (e.g. "anthropic/claude-3-haiku") |
| `prompt_version` | TEXT | NOT NULL, default 'v1' | Prompt template version for reproducibility |
| `summary` | TEXT | nullable | 2-3 sentence plain English description |
| `probable_cause` | TEXT | nullable | Root cause identified by the LLM |
| `recommendation` | TEXT | nullable | Human-readable recommended action |
| `recommended_action_id` | TEXT | nullable | Machine-readable action ID (matches policy) |
| `confidence_score` | FLOAT | CHECK 0.0–1.0 | LLM confidence in the analysis |
| `escalate` | BOOLEAN | default FALSE | Whether LLM recommends immediate human attention |
| `raw_output` | JSONB | nullable | Complete raw LLM response |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | When analysis was produced |

**Indexes:** `incident_id`

---

### `remediation_actions`
Record of every remediation action requested or executed.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `incident_id` | UUID | FK → incidents(id) ON DELETE CASCADE | Parent incident |
| `action_type` | TEXT | NOT NULL | Category (e.g. "deployment", "scaling") |
| `action_name` | TEXT | NOT NULL | Human-readable name |
| `requested_by` | TEXT | NOT NULL, default 'operator' | Who requested it |
| `execution_mode` | execution_mode | NOT NULL, default 'manual' | Manual or automatic |
| `status` | action_status | NOT NULL, default 'PENDING' | Current execution state |
| `parameters` | JSONB | default '{}' | Parameters passed to the playbook |
| `started_at` | TIMESTAMPTZ | nullable | When execution started |
| `completed_at` | TIMESTAMPTZ | nullable | When execution finished |
| `result_summary` | TEXT | nullable | One-line result description |
| `raw_output` | JSONB | nullable | Full Ansible stdout/stderr |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Row creation time |

**Indexes:** `incident_id`

---

### `audit_logs`
Immutable append-only log of all significant system events for compliance and debugging.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | UUID | PK | Unique identifier |
| `incident_id` | UUID | FK → incidents(id) ON DELETE SET NULL | Related incident (nullable) |
| `actor_type` | TEXT | NOT NULL, CHECK IN ('user', 'system', 'automation') | Who/what caused this event |
| `actor_id` | TEXT | NOT NULL, default 'system' | Specific actor identifier |
| `event_type` | TEXT | NOT NULL | Machine-readable event type (e.g. "incident.created") |
| `message` | TEXT | NOT NULL | Human-readable event description |
| `metadata` | JSONB | default '{}' | Additional structured context |
| `created_at` | TIMESTAMPTZ | NOT NULL, default NOW() | Event timestamp |

**Event types (examples):**
- `incident.created` — new incident opened
- `incident.status_changed` — status transition
- `enrichment.started` / `enrichment.completed`
- `analysis.started` / `analysis.completed` / `analysis.failed`
- `action.requested` — operator clicked Run
- `action.started` / `action.completed` / `action.failed`
- `notification.sent`

**Indexes:** `incident_id`, `created_at DESC`

---

## Entity Relationships

```
incidents (1) ──────── (N) alert_events
incidents (1) ──────── (N) incident_evidence
incidents (1) ──────── (N) analysis_results
incidents (1) ──────── (N) remediation_actions
incidents (1) ──────── (N) audit_logs
```

All child tables cascade-delete when the parent incident is deleted (except `audit_logs` which sets `incident_id` to NULL to preserve the audit trail).
